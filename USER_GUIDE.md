# 用户手册（管理后台）

面向**人**的操作手册：查询台怎么用、分析工作台怎么玩、写操作怎么审批。
架构与安全设计见 [DESIGN.md](DESIGN.md)，agent 接入见 [AGENT_GUIDE.md](AGENT_GUIDE.md)。

后台地址 `http://127.0.0.1:8100/admin`，用 `DBM_ADMIN_TOKEN` 登录。
左侧导航五页：**查询台 / Redis / 审批中心 / 操作审计 / 系统设置**（连接管理已并入系统设置）。

## 查询台（/admin/sql）

DataGrip 风格的 SQL IDE，日常"连库—看表—查数—改数—导出"一屏完成。

### 左栏（对象树）

- 连接下拉（含 ⚗ 分析工作区）→ 对象树（库 → tables → 表 → 列/键/索引，
  表名右侧显示容量 M/G/T）。双击表名直接开数据页；右键有菜单
  （打开数据 / SELECT / 统计行数 / DDL / 复制表名 / 导入到分析工作区 / DROP）。
- **Schema 管理**：库多时点树头「Schemas ▾」勾选只显示关心的 schema（可叠加文字筛选）。
- 下方有**历史**（按连接去重）、**SQL 片段库**（保存常用 SQL，点击载入）。

### 编辑器

- Monaco，⌘/Ctrl+Enter 运行，⌃Space 上下文补全（FROM 后补表、SELECT/WHERE 补列、
  `别名.` 补字段）。多条语句时只跑**光标所在那条**；选中则跑选区。
- **右键菜单**：① 格式化（有选中则只格式化选中，否则全部）② EXPLAIN（执行计划树）
  ③ EXPLAIN ANALYZE（实际执行并分析）。
- **函数文档 hover**：光标悬停在 MySQL 内置函数上，弹出中文说明 + 用法 + 官方文档链接
  （覆盖 179 个常用函数）。
- **语法标红**：实时按连接方言 sqlglot 检查，错误处红波浪。
- **右上角「执行 schema」**：MySQL/PG 可指定语句在哪个库/schema 下执行；选定后记为该连接
  默认，新 tab 自动沿用。

### 结果区

- **查询结果 tab**：每执行一次 SQL 新增一个结果 tab，可切换/关闭；**hover 结果 tab 显示对应
  的完整 SQL**。翻页只更新当前结果 tab。
- 分页（自动 LIMIT 兜底防大表拉挂库）、列头排序、「表格/图表」切换（柱/折线/饼/散点 +
  X/Y/聚合）、CSV/JSON/Markdown/XLSX 导出。
- **表头每列带类型图标**（# 数值 / T 字符串 / ◷ 时间日期 / {} JSON / ☑ 布尔 …），列名保持
  原样大小写。
- **Value / Record 面板**：点单元格右侧展开——Value 舒适编辑长文本/JSON（JSON 格式化、
  设为 NULL）、Record 整行视图（字段名 + 类型 + 值，JSON 自动美化）。**查询结果也支持**
  （只读查看 + 格式化，不可就地改）。
- **找东西**：⌘/Ctrl+F 结果内搜文本（高亮、Enter 跳转）；⌘/Ctrl+P 全局表名搜索（当前连接
  跨库），回车直达表数据。

### 表数据页（双击表打开）

- **WHERE / ORDER BY 条**走 SQL 重查（跨页正确），输入时**弹字段名提示下拉**（↑↓ 选、
  Enter/Tab 补全）；点列头排序。
- 下方是**可拖拽调高的「SQL 执行记录」面板**（本连接最近执行，过滤/排序的 SQL 即时进记录）。
- **列显示类型**（Change Display Type）：数值列右键可改为按 Timestamp（秒/毫秒/微秒）展示为
  日期 + 表头 🕒 徽章。**铁律：只是展示格式，底层值不变，编辑/提交写的永远是原值**。
- **时间列编辑用日期选择器**（date/datetime/time 各用对应控件）。

### 改数据：暂存式编辑 + 提交

在查询台改数据**不是逐条立即执行**，而是**先攒改动、再统一提交**：

- 双击单元格改值 → 单元格**标黄**；Value 面板保存同样是暂存。
- **数据工具栏**（过滤条上方）：**刷新**（有未提交改动会提醒会丢弃）、**加行**（网格顶部填一行，
  留空列用默认值；「克隆」以选中行为模板）、**减行**（标记选中行删除，整行删除线）、
  **提交（N）**、**DDL**（新 tab 看格式化建表语句）。
- **提交**两种：**SQL 确认后提交**（弹框展示本次所有变更的合并 SQL，确认后执行）/ **直接提交**。
  提交时由 writer 账号依次执行并审计——**管理员旁路，不进审批单**。Agent 的写操作没有这条路，
  必须走审批（见下）。
- 选中行可「复制为」TSV（贴 Excel）/ INSERT / Markdown / JSON。
- **数据导入**：表右键「导入数据（CSV/粘贴）…」——粘贴 CSV/TSV 或选文件，首行表头按列名
  匹配，预览确认后 writer 单事务导入。

### 环境防混淆（重要）

- 连接下拉与**每个 tab** 都带彩色环境徽章（PROD 红 / STAGING 橙 / DEV 蓝 / LOCAL 灰），
  一眼看清每个 tab 属于哪个连接、什么环境，**杜绝本地/线上混淆**。
- **连到生产库时整个查询台套红色边框 + 顶部红色警示条 + 连接框变红**。
- **切换连接会清掉该 tab 的旧结果与未提交改动**（数据 tab 转为普通查询 tab），线上数据不会
  残留在新连接标签下。

### 多 tab 与状态保活

- tab 分查询/数据/DDL/流程四类，可拖拽排序、⚲ 固定防误关。
- **全部状态保活**：关页面/刷新回来原样恢复；查询在服务端异步执行，切页不中断。持久化做了
  有界收敛（只存当前结果、超限分级降级），不会撑爆浏览器存储。

## Redis 控制台（/admin/redis，对标 Medis）

Redis 是键值模型，和 SQL 差异大，所以单独一页（左导航「Redis」），不混在查询台里。
Redis 连接**支持无认证**（本地无 auth 实例：连接管理里密码留空即可）。

- **左栏**：连接下拉 → **库**（底部切换器列逻辑库，有数据的库名后带键数、空库不带；
  库数很大的实例只列到 `max(16, 最大非空库号+1)`，避免刷屏，但有数据的库始终在）→
  **键**（按 `:` 前缀自动分组成文件夹，叶子键带 STRING/HASH/LIST/SET/ZSET 彩色类型徽章）。
  **点文件夹深度优先只展开「第一个」子文件夹这一条链、一直钻到底**（同层其他文件夹保持折叠，
  避免子项很多时刷屏）；再点一下收起整棵子树。顶部搜索框填匹配串（如 `offer:*`）回车按 SCAN 过滤。
- **命令窗口**（中间，Monaco 编辑器）：每行一条命令，**选中或光标所在行按 ⌘/Ctrl+Enter 执行**。
  读命令直接出结果；**写命令先弹风险确认条**，确认后由 writer 账号直接执行并审计
  （后台旁路，不进审批单——和查询台写操作一个道理）。**生产环境**的写命令除确认外，
  还须**输入连接名**匹配才放行（防误清线上库）。会回显凭证的命令（`CONFIG GET requirepass`、
  `ACL LIST/GETUSER` 等）结果中的密码/口令哈希一律脱敏。
- **键详情**（点左侧键）：类型感知展示——string 直接看值，hash/zset/list/set 以表格列出
  字段/成员/元素，附 TTL、内存占用、编码。
- **命令文档面板**（右侧）：把光标放到某条命令上，自动显示该命令的中文说明、语法，
  并链接到 redis.io 官方文档。覆盖 176 条常用命令。

**Redis 有意不暴露给 agent**（不在 MCP 工具里，也不出现在 `list_connections`）——
只有人能通过这个后台控制台操作 Redis。

## 分析工作台（跨源分析）

把不同库、不同连接、本地文件的数据快照进本地 DuckDB 沙箱自由分析。
详细设计与 agent 用法见 [ANALYSIS.md](ANALYSIS.md)。

1. **导入**：对象树里右键任意表 →「导入到分析工作区…」，填工作区名/数据集名/行数上限。
2. **分析**：连接下拉切到 ⚗ 工作区，编辑器里随便 JOIN/聚合/建 VIEW——沙箱内不需要审批。
3. **出图**：结果区切「图表」。

### Workflow：沉淀一套分析

- **脚本式**：在工作区写好多语句脚本 → 点「存工作流」。保存的内容 =
  取数配方（每个数据集从哪拉的，自动记录）+ 脚本 + 图表配置。
- **可视化 DAG（画布）**：左栏工作流区点「＋流程」→ 工具条加节点
  （取数/文件/过滤/JOIN/聚合/SQL/输出）→ **拖节点右缘绿点到下一节点左缘蓝点**连线
  → 点节点在右侧面板配置 →「运行流程」。逐节点标 ✓/✗；中间节点就是工作区里的视图，
  配置面板「预览数据」可单看。JOIN 节点左/右输入口在 SQL 里是别名 `l` / `r`；
  SQL 节点直接用上游**节点名**当表名。
- **重跑**：左栏工作流条目点 ▶ ——重新从源库拉最新数据 → 按序执行 → 输出
  （存过图表配置就直接出图）。点条目本身则载入编辑。
- 内置示例「示例 · 渠道ROI分析」演示了全部节点类型，可直接 ▶ 看效果。
- Agent 也能跑：对它说"跑一下 xx workflow 并解读结果"即可（`run_workflow`）。

## 审批中心（agent 写操作的闸门）

Agent 提交的数据变更会在这里排队：每单含 SQL 原文、风险报告（等级/影响表/行量级/
是否命中索引/理由）。**批准**后 agent 带 change_id 重提才真正执行（writer 账号）；
**拒绝**可填理由回传给 agent。审批单 30 分钟过期、一次性核销。
兜底 CLI：`dbm approvals` / `dbm approve <id>` / `dbm reject <id>`。

## 操作审计（/admin/audit）

每条 SQL/Redis 命令的完整留痕：agent 身份、连接、SQL、风险等级、结果行数/耗时/状态、
审批信息。支持按项目/连接/agent/状态筛选、自动刷新；默认隐藏 admin-ui 自己的操作
（可切换显示）。

## 系统设置（/admin/settings）

多 tab，服务端保存、跨浏览器一致：

- **整体设置**：界面主题 深色（默认）/ 浅色——选浅色后查询台与 Redis 两个 IDE 页变白底深字。
- **DB**：查询台结果每页行数（默认 100）。
- **Redis**：结果每页行数（默认 100，键详情集合与命令结果分页）+ 键列表加载上限（默认 1000，SCAN 一次最多加载）。
- **连接管理**：新增/编辑连接（弹窗表单）——引擎、主机、库、reader/writer 双账号、SSH 跳板链。
  页面输入的密码一律写系统 keyring，配置文件只存引用（`keyring://`）；
  支持「测试连接」「测试 SSH」和账号权限校验（reader 是否真只读）。连接改动热生效，无需重启。

主题/分页类设置改完保存后，重新打开查询台 / Redis 页生效。
