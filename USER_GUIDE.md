# 用户手册（管理后台）

面向**人**的操作手册：查询台怎么用、分析工作台怎么玩、写操作怎么审批。
架构与安全设计见 [DESIGN.md](DESIGN.md)，agent 接入见 [AGENT_GUIDE.md](AGENT_GUIDE.md)。

后台地址 `http://127.0.0.1:8100/admin`，用 `DBM_ADMIN_TOKEN` 登录。
左侧导航四页：**查询台 / 审批中心 / 操作审计 / 连接管理**。

## 查询台（/admin/sql）

DataGrip 风格的 SQL IDE，日常"连库—看表—查数—导出"一屏完成。

### 基本操作

- **左栏**：连接下拉（含 ⚗ 分析工作区）→ 对象树（库 → tables → 表 → 列/键/索引，
  表名右侧显示容量 M/G/T）。双击表名直接开数据页；右键有菜单
  （打开数据 / SELECT / 统计行数 / DDL / 复制表名 / 导入到分析工作区 / DROP）。
- **编辑器**：Monaco，⌘/Ctrl+Enter 运行，⌃Space 补全（上下文感知：FROM 后补表、
  SELECT/WHERE 补列、`别名.` 补字段）。多条语句时只跑**光标所在那条**；选中则跑选区。
- **右上角「执行 schema」**：MySQL/PG 可指定语句在哪个库/schema 下执行。
- **结果区**：分页（自动 LIMIT 兜底防大表拉挂库）、列头排序、
  「表格/图表」切换（柱/折线/饼/散点 + X/Y/聚合）、CSV/JSON/Markdown/XLSX 导出。
- **表数据页**（双击表打开）：WHERE 条 + ORDER BY 表达式走 SQL 重查（跨页正确）；
  双击单元格就地编辑；点单元格右侧 Value 面板（长文本/JSON 舒适编辑、
  JSON 格式化、设为 NULL、Record 整行视图）——保存生成按主键定位的 UPDATE，
  走写确认流。
- **行级增删**：点行号选择行（⇧范围 / ⌘多选）；「＋ 新增行」在网格里填一行生成
  INSERT（留空列用默认值）；「克隆」以选中行为模板；「删除」按主键生成 DELETE——
  全部走写确认流。选中行可「复制为」TSV（直接贴 Excel）/ INSERT / Markdown / JSON。
- **数据导入**：表右键「导入数据（CSV/粘贴）…」——粘贴 CSV/TSV（Excel 里复制的
  区域直接粘）或选文件，首行表头按列名自动匹配，解析预览确认后 writer 单事务导入。
- **找东西**：⌘/Ctrl+F 在结果网格里搜文本（命中高亮、Enter 跳转）；
  ⌘/Ctrl+P 全局表名搜索（当前连接跨库），回车直达表数据。
- **语法标红**：编辑器实时按连接方言做 sqlglot 语法检查，错误处红波浪。
- **环境识别与生产警示**：连接下拉每项带彩色环境徽章（PROD 红 / STAGING 橙 / DEV 蓝 / LOCAL 灰）。
  **连到生产库时整个查询台会套红色边框 + 顶部红色警示条 + 连接框变红**，一眼可辨、防误操作。
- **Schema 管理**：库多时点树头「Schemas ▾」勾选只显示关心的 schema（可叠加文字筛选）；
  右上角选定「执行 schema」后会记为该连接的默认，新开的查询 tab 自动沿用。
- **多 tab**：查询/数据/DDL/流程四类，可拖拽排序、⚲ 固定防误关。
  **全部状态保活**：关页面/刷新回来原样恢复；查询在服务端异步执行，切页不中断。
- 历史（按连接去重）、SQL 片段库（保存常用 SQL，点击载入）在左栏下方。

### 写操作（后台旁路）

在查询台跑 INSERT/UPDATE/DELETE/DDL：先弹**风险报告**（影响表/行量级/是否命中索引），
确认后由 writer 账号直接执行并审计——这是管理员旁路，**不进审批单**。
Agent 的写操作没有这条路，必须走审批（见下）。

## 分析工作台（跨源分析）

把不同库、不同连接、本地文件的数据快照进本地 DuckDB 沙箱自由分析。
详细设计与 agent 用法见 [ANALYSIS.md](ANALYSIS.md)。

1. **导入**：对象树里右键任意表 →「导入到分析工作区…」，填工作区名/数据集名/行数上限。
2. **分析**：连接下拉切到 ⚗ 工作区，编辑器里随便 JOIN/聚合/建 VIEW——沙箱内不需要审批。
3. **出图**：结果区切「图表」。

### Workflow：沉淀一套分析

- **脚本式**：在工作区写好多语句脚本 → 点「存工作流」。保存的内容 =
  取数配方（每个数据集从哪拉的，自动记录）+ 脚本 + 图表配置。
- **可视化 DAG（画布）**：左栏工作流区点「＋流程」→ 工具条加节点
  （取数/文件/过滤/JOIN/聚合/SQL/输出）→ **拖节点右缘绿点到下一节点左缘蓝点**连线
  → 点节点在右侧面板配置 →「运行流程」。逐节点标 ✓/✗；中间节点就是工作区里的视图，
  配置面板「预览数据」可单看。JOIN 节点左/右输入口在 SQL 里是别名 `l` / `r`；
  SQL 节点直接用上游**节点名**当表名。
- **重跑**：左栏工作流条目点 ▶ ——重新从源库拉最新数据 → 按序执行 → 输出
  （存过图表配置就直接出图）。点条目本身则载入编辑。
- 内置示例「示例 · 渠道ROI分析」演示了全部节点类型，可直接 ▶ 看效果。
- Agent 也能跑：对它说"跑一下 xx workflow 并解读结果"即可（`run_workflow`）。

## 审批中心（agent 写操作的闸门）

Agent 提交的数据变更会在这里排队：每单含 SQL 原文、风险报告（等级/影响表/行量级/
是否命中索引/理由）。**批准**后 agent 带 change_id 重提才真正执行（writer 账号）；
**拒绝**可填理由回传给 agent。审批单 30 分钟过期、一次性核销。
兜底 CLI：`dbm approvals` / `dbm approve <id>` / `dbm reject <id>`。

## 操作审计（/admin/audit）

每条 SQL/Redis 命令的完整留痕：agent 身份、连接、SQL、风险等级、结果行数/耗时/状态、
审批信息。支持按项目/连接/agent/状态筛选、自动刷新；默认隐藏 admin-ui 自己的操作
（可切换显示）。

## 连接管理（/admin/connections)

新增/编辑连接（弹窗表单）：引擎、主机、库、reader/writer 双账号、SSH 跳板链。
页面输入的密码一律写系统 keyring，配置文件只存引用（`keyring://`）；
支持「测试连接」「测试 SSH」和账号权限校验（reader 是否真只读）。
连接改动热生效，无需重启服务。
